# Redmine 일감 수정 스킬

일감 번호를 기준으로 Redmine 일감의 필드를 수정합니다.

---

## 1단계: 설정 확인

아래 환경변수가 설정되어 있는지 확인합니다.

| 환경변수 | 설명 | 필수 |
|----------|------|------|
| `REDMINE_URL` | Redmine 서버 URL | 필수 |
| `REDMINE_API_KEY` | Redmine API 키 | 필수 |

없으면 `.claude/settings.local.json`의 `env` 섹션에 추가해 달라고 안내하고 중단합니다.

---

## 2단계: 일감 조회

일감 번호가 없으면 사용자에게 확인합니다. 번호가 있으면 현재 상태를 조회합니다.

```bash
curl -s \
  -H "X-Redmine-API-Key: {api_key}" \
  "{redmine_url}/issues/{issue_id}.json"
```

조회 결과를 아래 형식으로 출력합니다.

```
## 현재 일감 상태 (#1234)

| 항목 | 현재 값 |
|------|---------|
| 제목 | 현재 제목 |
| 내용 | 현재 내용 (앞 100자) |
| 상태 | 진행 |
| 시작일 | 2026-04-20 |
| 완료기한 | (미지정) |
| 추정시간 | (미지정) |
| 진척도 | 0% |
```

---

## 3단계: 수정 항목 추출

사용자 입력에서 수정할 필드를 추출합니다.

| 필드 | Redmine 파라미터 | 허용값 |
|------|-----------------|--------|
| 제목 | `subject` | 텍스트 |
| 내용 | `description` | 텍스트 |
| 상태 | `status_id` | 진행(10) / 실적(12) |
| 시작일 | `start_date` | YYYY-MM-DD |
| 완료기한 | `due_date` | YYYY-MM-DD |
| 추정시간 | `estimated_hours` | 숫자 (시간 단위) |
| 진척도 | `done_ratio` | 0–100 |

### 추출 규칙

- **날짜**: "오늘", "이번 주 금요일" 등 상대 표현은 오늘 날짜 기준으로 변환.
- **진척도**: "50%", "절반" → 50 / "완료" → 100.
- **상태**: "진행" → 10 / "실적" → 12.
- **언급 없는 필드**: 변경하지 않음. API 페이로드에서 제외.

---

## 4단계: 미리보기 출력

수정 내용을 아래 형식으로 보여주고 확인을 받습니다.

```
## Redmine 일감 수정 미리보기 (#1234)

| 항목 | 현재 값 | 변경 값 |
|------|---------|---------|
| 제목 | 기존 제목 | 새 제목 |
| 상태 | 진행 | 실적 |
| 완료기한 | (미지정) | 2026-04-30 |
| 진척도 | 0% | 100% |

수정하시겠습니까? (확인 / 수정할 항목 말씀해 주세요)
```

변경되는 항목만 표에 표시합니다. 사용자가 추가 수정을 요청하면 해당 항목만 업데이트 후 미리보기를 다시 출력합니다.

---

## 5단계: API 호출

사용자가 확인하면 curl로 Redmine REST API를 호출합니다.

```bash
curl -s -X PUT \
  -H "X-Redmine-API-Key: {api_key}" \
  -H "Content-Type: application/json" \
  -d '{
    "issue": {
      "subject": "{subject}",
      "description": "{description}",
      "status_id": {status_id},
      "start_date": "{start_date}",
      "due_date": "{due_date}",
      "estimated_hours": {estimated_hours},
      "done_ratio": {done_ratio}
    }
  }' \
  "{redmine_url}/issues/{issue_id}.json"
```

- 변경하지 않는 필드는 JSON 페이로드에서 제외합니다.
- **한글 등 멀티바이트 문자는 반드시 유니코드 이스케이프(`\uXXXX`)로 변환 후 전송합니다.** 서버 WAF가 멀티바이트 문자를 직접 차단합니다.
- 성공 시 HTTP 200 반환.

### 성공 응답 처리

```
✅ 일감이 수정되었습니다.
- 일감 번호: #1234
- URL: {redmine_url}/issues/1234
```

### 실패 응답 처리

HTTP 상태 코드와 응답 본문을 사용자에게 그대로 보여주고, 가능한 원인을 한 줄로 설명합니다.
