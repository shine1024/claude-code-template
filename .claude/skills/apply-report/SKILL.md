## 목적

`/analyze-report` 가 생성한 보고서(`reports/analyze-report/{날짜}-analyze-report.md`)의 규칙 후보를 검토하여
사용자가 채택한 항목만 프로젝트의 규칙 파일에 반영합니다.

| 보고서 섹션 | 적용 대상 | 비고 |
|------------|----------|------|
| `## 신규 발견 — 규칙 개선 제안` | ✓ | 태그·규칙 타입에 따라 적용 위치 결정 |
| `## 개인 규칙 후보 승격 검토` | ✓ | 강도 `강`/`중`만 기본 노출, `약`은 옵션 |
| `## 사용자 가이드 제안` | ✗ | 적용 대상 아님 — 미리보기에 참고용으로만 표시 |
| `## 정보 보완 필요 항목` | ✗ | 적용 대상 아님 — 사용자에게 별도 안내 |

> 보고서 본문을 Claude 가 다시 해석하지 않습니다. `parse-report.js` 가 표를 파싱해 JSON 으로 출력합니다 — 컨텍스트 절약 목적.

---

## 1단계: 보고서 선택

인자로 날짜를 받거나(`/apply-report 2026-04-30`), 인자가 없으면 가장 최근 보고서를 자동 선택합니다.

```bash
node "{SKILL_DIR}/scripts/parse-report.js" [YYYY-MM-DD]
```

> `SKILL_DIR` 은 스킬 실행 시 상단에 표시되는 Base directory 경로입니다.

보고서가 없으면 아래 메시지를 출력하고 종료합니다.

```
분석 보고서가 없습니다. /analyze-report 를 먼저 실행하세요.
```

스크립트 출력 JSON 의 `reportPath` 를 사용자에게 알리고 다음 단계로 진행합니다.

---

## 2단계: 후보 정리 및 ID 부여

파싱 결과(`newRules`, `promotions`)에서 적용 가능한 후보만 추려 일련번호(`#1`, `#2`, ...)를 부여합니다.

**제외 기준**
- `[사용자-가이드]` 태그 → 적용 대상 아님 (4단계에서 별도 안내)
- 개인 규칙 후보 중 강도 `약` → 기본 제외 (사용자가 "약도 포함" 요청 시에만 노출)

**규칙 타입별 기본 적용 위치 매핑** (보고서의 `적용 위치(추정)` 가 비어있을 때 사용)

| 규칙 타입 | 기본 위치 |
|----------|----------|
| `[코딩규칙]` | `.claude/rules/{lang}-style.md` (lang 추정 어려우면 루트 `CLAUDE.md`) |
| `[탐색규칙]` | 루트 `CLAUDE.md` 또는 `.claude/rules/{도메인}.md` |
| `[지식]` | 해당 모듈 `CLAUDE.md` |
| `[워크플로우]` | 루트 `CLAUDE.md` > AI 협업 지침 |

보고서에 `적용 위치(추정)` 이 명시되어 있으면 그 값을 우선합니다.

---

## 3단계: 미리보기 + 선택

후보 목록을 사용자에게 표 형식으로 보여주고 선택을 받습니다.

```
## 적용 후보 (보고서: reports/analyze-report/{날짜}-analyze-report.md)

| # | 종류  | 태그/강도   | 규칙 타입  | 요약                | 적용 위치              |
|---|------|------------|-----------|---------------------|----------------------|
| 1 | 신규 | [규칙-누락] | [코딩규칙] | ...                 | ./CLAUDE.md          |
| 2 | 신규 | [규칙-미적용] | [워크플로우] | ...               | ./CLAUDE.md          |
| 3 | 승격 | 강          | [지식]     | ...                 | ./modules/pce/CLAUDE.md |

## 적용 대상 아님 (참고용)

- 사용자 가이드: N건 — 보고서의 "사용자 가이드 제안" 섹션 참고
- 정보 보완 필요: M건 — 보고서의 "정보 보완 필요 항목" 섹션 참고

적용할 항목 번호를 입력하세요 (예: 1,2,3 / all / 취소):
```

**입력 처리**
- `1,3,5` → 해당 번호만 적용
- `all` → 표시된 모든 후보 적용
- `취소` 또는 빈 입력 → 종료
- 약 강도 포함 요청 (`약도 포함`) → 표를 다시 보여주고 재입력 받음

---

## 4단계: 멱등성 체크

선택된 항목들의 적용 위치 파일을 `Read` 로 읽어 본문에 같은 내용이 이미 있는지 확인합니다.

비교 기준: 규칙 본문의 핵심 키워드(첫 문장 또는 헤딩)가 대상 파일에 이미 존재하면 `이미 적용됨` 으로 처리하고 사용자에게 알립니다.

```
#2 는 이미 ./CLAUDE.md 에 동일 내용이 있어 건너뜁니다.
```

이미 적용된 항목은 적용 단계에서 제외합니다.

---

## 5단계: 적용 (Edit)

각 항목을 적용 위치 파일에 `Edit` 로 추가합니다.

**삽입 위치 결정 가이드**

| 규칙 타입 | 권장 삽입 위치 |
|----------|---------------|
| `[코딩규칙]` | 해당 언어 스타일 파일의 가장 적합한 섹션(네이밍·포맷·기타 등). 없으면 끝에 추가 |
| `[탐색규칙]` | `CLAUDE.md` 의 `## 코드 탐색` / 관련 도메인 규칙 파일 |
| `[지식]` | 모듈 `CLAUDE.md` 의 `## 프로젝트 개요` 또는 도메인 섹션 |
| `[워크플로우]` | 루트 `CLAUDE.md` 의 `## AI 협업 지침` 또는 `## 하지 말아야 할 것` |

**작성 원칙**
- 보고서의 `제안 내용` 또는 `규칙 요약` 을 그대로 옮기되, 대상 파일의 기존 문체·들여쓰기·헤딩 깊이에 맞춰 다듬습니다.
- 같은 섹션 내에 비슷한 주제의 항목이 있으면 인접 위치에 삽입합니다.
- 헤딩(`###`) 추가는 신규 주제가 명확할 때만 — 그 외에는 기존 헤딩 아래에 불릿/문단으로 추가합니다.
- 사용자 가이드/정보 보완은 적용하지 않습니다.

각 항목 적용 후 어떤 파일의 어디에 무엇을 추가했는지 한 줄로 기록해 둡니다 (마지막 요약과 CHANGES.md 작성에 사용).

---

## 6단계: CLAUDE.local.md 정리

적용된 규칙과 매칭되는 `## 공유 가능` 블록을 찾아 사용자 확인 후 제거합니다.

```bash
echo '[{"id":1,"text":"적용된 규칙 본문..."}, ...]' | node "{SKILL_DIR}/scripts/find-local-matches.js"
```

> 입력은 5단계에서 적용한 각 규칙의 본문이며, `id` 는 적용 항목 번호입니다.

스크립트는 매칭 후보(score ≥ 0.4)를 출력합니다. 사용자에게 다음과 같이 보여줍니다.

```
## CLAUDE.local.md 공유 가능 섹션 정리 후보

다음 블록은 방금 적용한 규칙과 유사도가 높습니다 (중복 가능성).

| 블록ID | 매칭된 규칙# | 유사도 | 블록 본문 (앞 80자) |
|--------|------------|--------|---------------------|
| 2      | #1         | 0.85   | ...                 |
| 5      | #3         | 0.62   | ...                 |

제거할 블록ID 를 입력하세요 (예: 2,5 / all / 건너뛰기):
```

**입력 처리**
- `2,5` → 해당 블록만 제거
- `all` → 표시된 모든 매칭 블록 제거
- `건너뛰기` 또는 빈 입력 → 정리 단계 생략

확인된 블록ID 를 다음 스크립트에 전달하여 제거합니다.

```bash
echo '{"blockIds":[2,5]}' | node "{SKILL_DIR}/scripts/clean-local-rules.js"
```

> `## 비공유` 섹션은 절대 건드리지 않습니다.
> 제거 후 인접 빈 줄이 3줄 이상이면 1줄로 자동 정리됩니다.

매칭 후보가 0건이면 정리 단계를 자동 건너뜁니다.

## 7단계: CHANGES.md 갱신

`CHANGES.md` 상단에 오늘 날짜 그룹으로 항목을 추가합니다.

- 카테고리: 규칙 추가는 `[수정]`, 신규 섹션·파일 생성은 `[기능]`
- 형식: `- [수정] CLAUDE.md / .claude/rules/* — apply-report 적용 (보고서: {YYYY-MM-DD})`
- 적용 항목별로 한 줄씩 들여쓰기 불릿으로 부연 설명

> 같은 날짜에 이미 항목이 있으면 그룹 헤딩에 `(N)` 을 붙여 새 그룹을 만들거나, 같은 흐름의 후속이면 기존 항목에 병합합니다 — 프로젝트 `CLAUDE.md` 의 변경 시 체크리스트 참고.

---

## 8단계: 결과 요약

마지막으로 한 화면 요약을 출력합니다.

```
적용 완료

- 적용된 규칙: N건
  - #1 → ./CLAUDE.md (## AI 협업 지침)
  - #3 → ./.claude/rules/sql-style.md (## 네이밍)
- 건너뜀(이미 적용): K건
- CLAUDE.local.md 정리: M건
- CHANGES.md 갱신: 완료

변경 확인: git diff
커밋은 사용자가 직접 진행해주세요. (자동 커밋 안 함)
```

---

## 주의 사항

- **커밋·푸시는 절대 자동으로 하지 않습니다** — 변경 후 `git diff` 안내만 합니다.
- **`## 비공유` 섹션은 어떤 단계에서도 건드리지 않습니다.**
- **사용자 가이드 / 정보 보완 항목은 적용하지 않습니다** — 보고서 본문을 사용자가 직접 읽도록 안내만 합니다.
- 보고서가 너무 오래되어 적용 위치 파일이 이미 변경되었을 수 있습니다 — 5단계 멱등성 체크에서 걸러지지만, 의심되면 사용자에게 확인합니다.
- 약 강도(`약`) 후보는 1명만 제안한 항목이라 신중히 — 기본 제외이며 사용자 명시 요청 시에만 노출합니다.
