# 문서 작성 규칙

`.claude/skills/<name>/SKILL.md`, 가이드, README 등 사용자·Claude가 참조하는 문서를 작성할 때의 표준.

## SKILL.md 표준 구조

### Frontmatter (필수)
```
---
name: skill-name
description: 한 줄 설명. Claude가 이 스킬을 언제 호출할지 판단하는 기준이 되므로 트리거 키워드를 포함한다.
---
```

- `description`은 사용자가 어떤 말을 했을 때 이 스킬이 호출되어야 하는지를 적는다
- 가능하면 트리거 예시(`"X 해줘"`, `"/skill-name"`)를 description 또는 본문에 포함한다

### 본문 섹션
다음 순서를 권장하되, 영역에 맞지 않는 섹션은 생략한다.

1. 개요 — 무엇을 하는 스킬인가 (2~4문장)
2. 사용 시점 — 언제 호출되는가, 트리거 표현
3. 실행 절차 — 단계별 동작 (번호 또는 체크리스트)
4. 예시 — 실제 입력·출력·명령어. 복사해서 그대로 실행 가능해야 한다
5. 설정 / 환경변수 — 필요한 사전 설정 (있으면)
6. 제한 사항 — 동작하지 않는 케이스, 알려진 제약

## 작성 원칙
- 짧고 구체적으로 — Claude는 긴 설명보다 명확한 단계를 더 잘 따른다
- 코드·명령어 예시는 그대로 복사 가능해야 한다 (placeholder는 `<>`로 표시)
- 한국어 작성 시 명령형 어미 통일 ("~한다", "~하지 않는다")
- 이모지·장식 문자는 사용자가 명시적으로 요청한 경우에만 사용

## 금지
- 동일한 내용을 여러 파일에 중복 작성 (참조로 대체)
- 검증되지 않은 동작을 사실처럼 작성
- 사내 시스템·계정·로컬 경로 등 비공유 정보를 공용 가이드에 직접 기재
