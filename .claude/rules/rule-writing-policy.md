# 규칙 작성 정책

세션 중 새 규칙이 발견되면 `CLAUDE.local.md`의 `RULE_MODE` 설정에 따라 아래와 같이 동작한다.

---

## RULE_MODE 읽는 방법

아래 순서로 확인하고 먼저 발견된 값을 사용한다.

1. `CLAUDE.local.md`의 `## 규칙 작성 모드` — 개인 오버라이드 (우선)
2. `CLAUDE.md`의 `## 규칙 작성 모드` — 팀 기본값

두 파일 모두 설정이 없으면 기본값 `local`로 동작한다.

---

## RULE_MODE: direct

CLAUDE.md에 바로 반영하는 방식. 팀 전체가 동일한 규칙을 즉시 공유하는 프로젝트에 적합하다.

**동작**
- 새 규칙 발견 시 → CLAUDE.md 추가 내용을 제안한다
- 사용자 확인 후에만 CLAUDE.md에 작성한다 (git 커밋 대상이므로 반드시 확인)
- 확인 없이 CLAUDE.md를 임의로 수정하지 않는다

---

## RULE_MODE: local

CLAUDE.local.md에 먼저 기록하는 방식. 개인이 발견한 패턴을 축적한 뒤 선별적으로 팀에 공유하는 프로젝트에 적합하다.

**동작**
- 새 규칙 발견 시 → 아래 기준으로 섹션을 판단하여 CLAUDE.local.md에 자동 추가한다
- 팀 공유는 `/share-rules` 스킬로 수동 진행한다

### 공유 가능 판단 기준 (`## 공유 가능`)

아래 중 하나에 해당하면 `## 공유 가능`에 추가한다.

- 코딩 스타일, 네이밍 규칙
- 코드 탐색·작업 순서 패턴
- 도메인 업무 규칙 (특정 환경에 종속되지 않는 것)
- 팀원 누구에게도 적용 가능한 내용

### `## 공유 가능` 작성 포맷

각 규칙은 반드시 `###` 헤딩으로 시작하고 본문을 바로 아래에 작성한다.

```
### 규칙 제목
규칙 본문
```

헤딩 없이 본문만 작성하면 `/share-rules` 업로드 시 앞 규칙에 병합될 수 있다.

### 비공유 판단 기준 (`## 비공유`)

아래 중 하나에 해당하면 `## 비공유`에 추가한다.

- 로컬 파일 경로, 드라이브 경로
- 계정 정보, 사번, API 키
- 사내 URL, 내부 서버 주소
- 이 개발 환경에서만 유효한 설정
