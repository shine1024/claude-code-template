# CLAUDE-TEMPLATE.md

- 이 파일은 프로젝트의 `CLAUDE.md` 초안을 작성하기 위한 표준 템플릿입니다.
- `[ ]` 항목을 실제 프로젝트 정보로 채워주세요.
- 불필요한 섹션은 삭제해도 됩니다.
- **전체 200줄 이내를 권장합니다.** (초과 시 Claude의 컨텍스트 로딩 효율이 저하되어 불필요한 노이즈가 발생할 수 있습니다.)

---

## 사용 방법

1. 이 파일을 참고하여 프로젝트 루트에 `CLAUDE.md`를 생성합니다.
2. 멀티 모듈 프로젝트는 각 모듈에 `CLAUDE.md`를 추가 작성합니다.
3. Claude가 틀린 결과를 낼 때마다 관련 내용을 추가·수정합니다.

> **IMPORTANT — 작업 완료 기준**: 아래 체크리스트를 **모두 통과**한 후에야 작업 완료로 보고하십시오.
> 체크리스트 항목 중 하나라도 미완료인 경우 작업을 계속 진행하십시오.

## 작성 체크리스트

- [ ] 루트 CLAUDE.md가 200줄 이내인가?
- [ ] 아키텍처 패턴(레이어 구조, 트랜잭션, 베이스 클래스)을 작성했는가?
- [ ] 단일 모듈 프로젝트인 경우, 루트 CLAUDE.md의 '4. 코딩 컨벤션' 섹션에 rules/ 파일을 참조하도록 작성했는가?
- [ ] **멀티 모듈 프로젝트인 경우, 각 모듈 디렉터리에도 CLAUDE.md를 생성했는가?**
  - 멀티 모듈 여부는 `pom.xml`의 `<modules>` 태그 또는 `settings.gradle`의 `include` 구문으로 확인합니다.
  - 루트 CLAUDE.md만 생성하고 모듈 CLAUDE.md를 생성하지 않은 경우 작업 미완료입니다.
  - 각 모듈 CLAUDE.md는 하단 **멀티 모듈 CLAUDE.md 가이드**를 따라 작성합니다.

## 5. Claude 작업 지침

> 아래 지침은 Claude가 코드 생성·수정 시 반드시 준수해야 하는 행동 원칙입니다.
> 프로젝트 특성에 맞게 항목을 추가·수정하세요.

- 하나의 요청에 여러 기능이 포함된 경우, 기능 단위로 나눠 순차 진행한다
- 프로젝트에 존재하지 않는 함수·API를 임의로 생성하지 않는다
  → 불확실한 경우 기존 코드에서 실제 사용 중인 패턴을 먼저 탐색한다
- "이 프로젝트는 ~한 패턴을 사용한다"고 판단하기 전 전체 파일을 확인한다
  → 샘플 2~3개 기준으로 전체를 일반화하지 않는다
- 신규 화면 개발 시 기존 유사 화면을 참조하여 UI 패턴·CSS·JS 구조를 맞춘다
  → 참조 파일은 사용자가 명시하거나, 없으면 가장 유사한 기존 화면을 먼저 탐색한다
- 외부 라이브러리 API 탐색 시 내부 구현이 라이브러리 호출인 경우 공식 문서를 참조한다

# 멀티 모듈 CLAUDE.md 가이드

## 레벨별 작성 원칙

| 레벨           | 담을 내용                               |
| -------------- | --------------------------------------- |
| 루트 CLAUDE.md | 프로젝트 개요, 모듈 구조, 아키텍처 패턴 |
| 모듈 CLAUDE.md | 모듈 역할, 폴더 구조, 모듈 전용 컨벤션  |
| rules/\*.md    | 언어별 공통 코드 스타일 (자동 참조)     |

> 모듈 CLAUDE.md에서는 루트 내용을 반복하지 않습니다.
> 공통 코딩 규칙은 `.claude/rules/java-style.md` / `js-style.md` / `sql-style.md`를 참조하며, 각 모듈 CLAUDE.md에는 해당 모듈의 특화 규칙만 작성합니다.

## 모듈 CLAUDE.md 예시

```markdown
# [모듈명] — CLAUDE.md

> 공통 사항은 루트 CLAUDE.md 참고

## 모듈 역할

> 이 모듈이 하는 일 한 줄 설명

## 폴더 구조

> 이 모듈의 핵심 폴더만

## 모듈 전용 컨벤션

> 공통 규칙: `.claude/rules/java-style.md` / `js-style.md` / `sql-style.md` 참고
> 해당 모듈의 특화된 규칙을 [모듈명]-style.md 파일로 작성한다.
```
