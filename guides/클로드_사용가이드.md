## 1. Claude Code 소개

### Claude Code란?

Claude Code는 Anthropic이 만든 **에이전트 코딩 도구**입니다. 단순히 질문에 답하는 챗봇이 아니라, 실제로 파일을 읽고, 코드를 수정하고, 명령을 실행하며, 문제를 자율적으로 해결하는 AI 개발 파트너입니다.

```
Claude Code = AI 두뇌 + 파일 읽기/편집 + 명령 실행 + Git 연동 + 외부 도구 연결
```

**핵심 특징:**
- 코드베이스 전체를 이해하고 여러 파일에 걸친 작업 수행
- 파일 읽기/편집, 터미널 명령 실행, Git 작업 자동화
- 자율적으로 계획하고, 구현하고, 검증
- 여러분이 자리를 비운 동안에도 작업 수행 가능

### 사용 가능한 환경

| 환경 | 특징 |
|------|------|
| **터미널 CLI** | 가장 완전한 기능, 파이프/스크립트 연동 |
| **VS Code 확장** | 인라인 diff, @-멘션, Plan 검토 |
| **JetBrains 플러그인** | IntelliJ IDEA, WebStorm 등 지원 |
| **데스크톱 앱** | 시각적 diff, 병렬 세션, 예약 작업 |
| **웹 (claude.ai/code)** | 브라우저에서 로컬 설치 없이 사용 |

### 주요 기능 한눈에 보기

```
✅ 피처 개발 / 버그 수정 / 리팩토링
✅ 커밋 메시지 작성 / PR 생성
✅ 코드 탐색 및 설명
✅ CI/CD 파이프라인 연동
✅ MCP를 통한 외부 도구 연결 (Slack, Jira 등)
✅ 반복 작업 자동화 및 예약 실행
✅ 병렬 에이전트 팀 운영
```



### Plan Mode 활용

**Plan Mode**는 Claude가 파일을 수정하지 않고 분석과 계획만 수행하는 읽기 전용 모드입니다.

**전환 방법:** `Shift+Tab` 반복 → `⏸ plan mode on` 표시될 때까지

**4단계 권장 워크플로우:**

```
1️⃣ 탐색 (Plan Mode)
   → "AI 자동전표 추천 기능에 영향을 주는 파일들을 분석해줘."

2️⃣ 계획 (Plan Mode)
   → "RecommendAPI와 프론트엔드 병합 로직 개선 구현 계획을 작성해줘."
   → Ctrl+G로 계획을 편집기에서 열고 검토

3️⃣ 구현 (Normal Mode로 전환)
   → "계획대로 구현해줘."

4️⃣ 커밋
   → "refs #47188 형식으로 커밋해줘."
```

> 작은 단건 수정(오타 수정, 로그 추가 등)은 Plan Mode 없이 바로 진행하세요.

## 4. 메모리 관리 (CLAUDE.md)

### CLAUDE.md란?

Claude Code가 **매 세션 시작 시 자동으로 읽는** 마크다운 파일입니다. 프로젝트 규칙, 코딩 표준, 빌드 명령 등을 작성해두면 Claude가 항상 그 맥락을 가지고 작업합니다.

```
CLAUDE.md = Claude에게 보내는 지속적인 지시서
```

### CLAUDE.md 파일 배치 위치

| 범위 | 위치 | 적용 대상 | Git 공유 |
|------|------|-----------|----------|
| **조직 전체** | `C:\Program Files\ClaudeCode\CLAUDE.md` | 머신의 모든 사용자 | IT 관리자 배포 |
| **프로젝트** | `./CLAUDE.md` 또는 `./.claude/CLAUDE.md` | 팀 전체 | ✅ 커밋 권장 |
| **개인** | `~/.claude/CLAUDE.md` | 본인의 모든 프로젝트 | ❌ 개인 유지 |

### 효과적인 CLAUDE.md 작성법

**핵심 원칙:**
- **200줄 이하** 유지 (길수록 준수율 저하)
- **구체적으로** 작성 ("코드 정리" ❌ → "NSParam/NSReturn 사용 필수" ✅)
- **마크다운 헤더와 목록**으로 구조화
- **자주 검토**하여 오래된/충돌하는 지침 제거

**포함할 내용 vs 제외할 내용:**

| ✅ 포함 | ❌ 제외 |
|---------|---------|
| Maven 빌드/실행 명령 | 코드를 읽으면 알 수 있는 내용 |
| SAP RFC 연동 주의사항 | 자명한 관행 |
| AbstractJAVAService 상속 규칙 | 자주 변경되는 정보 |
| $nst 호출 패턴 | 상세한 RFC 문서 |
| 커밋 메시지 규칙 (refs #번호) | 표준 언어 규칙 |

### `/init` 으로 자동 생성

```bash
# 프로젝트 루트에서 실행하면 코드베이스를 분석해서 CLAUDE.md 초안 생성
/init
```

### `.claude/rules/` 규칙 파일 구성

대형 프로젝트에서 CLAUDE.md를 주제별로 분리:

```
unidocu6/
├── CLAUDE.md              # 주 프로젝트 지침 (간결하게)
└── .claude/
    └── rules/
        ├── frontend.md    # JS/$nst/$u 규칙
        ├── backend.md     # AbstractJAVAService/NSParam 규칙
        └── security.md    # 보안 요구사항
```

**경로별 규칙** (특정 파일에만 적용):

```markdown
---
paths:
  - "src/main/java/com/unipost/unidocu/ns/**/*.java"
---

# 서비스 개발 규칙
- AbstractJAVAService 상속 필수
- NSParam으로 입력, NSReturn으로 출력
- RFC 호출은 JCOManagerWrapper를 통해서만
```

### 자동 메모리 시스템

Claude는 세션을 통해 학습한 내용을 자동으로 저장합니다.

**저장 위치:** `~/.claude/projects/<project>/memory/`

```
memory/
├── MEMORY.md          # 인덱스 (매 세션에 자동 로드)
├── bug_all_recommendations.md   # 버그 패턴 기록
└── sap_rfc_conventions.md       # SAP 연동 결정사항
```

**Claude에게 직접 저장 요청:**
```
"IS_USER 구조체는 JCOManagerWrapper가 자동으로 채운다는 걸 기억해줘."
"OT_DATA 결과 파싱 시 배열 역직렬화에 주의해야 한다는 걸 기억해줘."
```

### UniDocu6용 CLAUDE.md 예시 ( /init 명령으로 초기화 이후 )

프로젝트의 `CLAUDE.md`는 이미 프로젝트 구조, 빌드 명령, 아키텍처 흐름, 컨벤션을 포함하고 있습니다. 팀 합의 하에 아래 항목을 추가로 관리하세요:

```markdown
## 자주 실수하는 패턴

- SAP RFC 호출 결과는 항상 NSReturn으로 받아 stringReturns/tableReturns 사용
- JS에서 $nst.is_data_ot_data(serviceId, params, callback) — OT_DATA가 테이블 결과
- 커밋 메시지 반드시 `refs #이슈번호` 접두사
- Maven 사용 (Gradle 아님): `mvn clean package`, `mvn tomcat7:run`
- D/B 연동은 SAP RFC (MyBatis/JPA 없음)
```

## 5. 모범 사례

### 원칙: 컨텍스트 윈도우가 핵심 자원

Claude의 성능은 컨텍스트 윈도우가 채워질수록 저하됩니다. 모든 모범 사례는 이 제약에서 비롯됩니다.

---

### 1️⃣ 작업 검증 방법을 제공하라

> Claude가 스스로 검증할 수 있을 때 결과 품질이 크게 향상됩니다.

| 전략 | 이전 | 이후 |
|------|------|------|
| 검증 기준 제공 | "RFC 호출 수정해줘" | "Z_FI_GET_ACCT RFC를 호출해서 HKONT 목록 반환하도록 수정. 빌드 후 `mvn tomcat7:run`으로 확인해줘." |
| 근본 원인 해결 | "빌드가 실패해" | "이 에러로 빌드 실패함: [에러]. 수정하고 `mvn clean package` 성공 확인해줘." |

---

### 2️⃣ 탐색 → 계획 → 코드 순서를 지켜라

```
❌ 잘못된 방식: "AI 추천 기능 추가해줘" → 즉시 코드 작성
✅ 올바른 방식: 탐색(Plan Mode) → 계획 검토 → 구현
```

**Plan Mode에서 새 세션 시작:**
```bash
claude --permission-mode plan
```

---

### 3️⃣ 구체적인 컨텍스트를 제공하라

```bash
# @로 파일 직접 참조
"@src/main/java/com/unipost/unidocu/ns/efi/ExpenseService.java 이 서비스의 RFC 호출 흐름을 설명해줘."

# 에러 로그 파이프
cat error.log | claude -p "이 에러의 근본 원인을 분석해줘."

# 스크린샷/이미지 붙여넣기 (Ctrl+V)
"[붙여넣은 UI 스크린샷] 이 화면과 같은 레이아웃을 구현해줘."
```

### 4️⃣ OpusPlan 모델 전략 (추천)

`opusplan`은 **계획에는 Opus, 실행에는 Sonnet**을 자동으로 전환하는 하이브리드 모드입니다.

```bash
# 세션 시작 시 설정
claude --model opusplan

# 또는 settings.json에서 기본값으로 설정
{ "model": "opusplan" }
```

| 단계 | 사용 모델 | 특징 |
|------|-----------|------|
| Plan Mode (계획/분석) | **Opus** | 복잡한 추론, 아키텍처 결정 |
| Normal Mode (코드 작성) | **Sonnet** | 빠른 구현, 효율적 |

**기타 모델 별칭:**

| 별칭 | 설명 |
|------|------|
| `opus` | 가장 강력한 모델 (복잡한 추론) |
| `sonnet` | 일반 코딩 작업 (권장) |
| `haiku` | 빠르고 간단한 작업 |
| `opusplan` | Plan: Opus / 실행: Sonnet 자동 전환 |

---

### 5️⃣ 노력 수준(Effort Level)을 조정하라

```bash
/effort low     # 간단한 작업 (빠르고 저렴)
/effort medium  # 일반 코딩 (기본값, 권장)
/effort high    # 복잡한 디버깅, 아키텍처 설계
/effort max     # 최고 깊이 (Opus 4.6 전용)
```

**일회성 깊은 추론:** 프롬프트에 `ultrathink` 포함
```
"이 SAP RFC 연동 오류의 근본 원인을 ultrathink로 분석해줘."
```

### 6️⃣ 세션을 능동적으로 관리하라

```bash
# 관련 없는 작업 사이에 컨텍스트 초기화
/clear

# 대화 압축 (중요한 부분 유지)
/compact

# 대화 요약 초점 지정
/compact "RFC 연동 변경사항만 유지"

# 이전 상태로 되돌리기
Esc+Esc  또는  /rewind
```

**빠른 질문은 컨텍스트에 넣지 않기:**
```
/btw  # 답변이 대화 히스토리에 포함되지 않는 사이드 질문
```

### 7️⃣ 일반적인 실패 패턴 피하기

| 패턴 | 문제 | 해결책 |
|------|------|--------|
| 주방 싱크 세션 | 관련 없는 작업들로 컨텍스트 오염 | 작업 간 `/clear` 실행 |
| 반복적 수정 | 실패한 접근 방식으로 오염 | 2번 실패 후 `/clear` + 더 나은 초기 프롬프트 |
| 과도한 CLAUDE.md | 중요한 규칙이 노이즈에 묻힘 | 무자비하게 정리, 200줄 이하 유지 |
| 무한 탐색 | 수백 개 파일 읽어 컨텍스트 낭비 | 탐색은 범위 지정 또는 Subagent 위임 |

### 개요: 설정 범위 계층


### Hooks 설정

Hooks는 Claude의 작업 특정 시점에 셸 명령을 자동 실행합니다.

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            // Java 파일 수정 시 파일명 로그 출력
            "command": "jq -r '.tool_input.file_path // empty' | grep -E '\\.java$' | head -1 | xargs -r -I{} sh -c 'echo \"파일 수정됨: {}\"'"
          }
        ]
      }
    ],
    "Notification": [
      {
        "matcher": "idle_prompt",
        "hooks": [
          {
            "type": "command",
            // Claude 작업 완료 시 Windows 팝업 알림
            "command": "powershell.exe -Command \"[System.Windows.Forms.MessageBox]::Show('Claude 작업 완료!', 'Claude Code')\""
          }
        ]
      }
    ]
  }
}
```

**주요 Hook 이벤트:**

| 이벤트 | 발생 시점 |
|--------|-----------|
| `PostToolUse` | 도구 사용 후 |
| `PreToolUse` | 도구 사용 전 |
| `Notification` | Claude가 입력 대기할 때 |
| `InstructionsLoaded` | 지침 파일 로드 시 |

> 세션 내에서 `/hooks`를 실행하면 설정된 hooks를 확인할 수 있습니다.

## 7. Skills 활용

### Skills란?

Skills는 Claude의 **재사용 가능한 워크플로우**입니다. 반복되는 작업을 Skill로 만들어두면 `/skill-name`으로 호출하거나, 관련 대화 시 자동으로 로드됩니다.

```
Skills = 팀 전용 작업 레시피 + 자동화 워크플로우
```

### 번들 Skills (기본 제공)

| Skill | 설명 |
|-------|------|
| `/batch <지시>` | 코드베이스 전체 대규모 변경을 병렬 에이전트로 조율 |
| `/debug [설명]` | 디버그 로그 활성화 및 문제 분석 |
| `/loop [간격] <프롬프트>` | 프롬프트 반복 실행 (빌드 폴링 등) |
| `/simplify [초점]` | 최근 변경 코드 품질 리뷰 및 개선 |

### skill-creator로 Skills 만들기

기존 Skill을 직접 작성하는 것보다 **`skill-creator` 플러그인**을 활용하면 훨씬 빠르게 양질의 Skill을 만들 수 있습니다.

**skill-creator 호출:**
```
/skill-creator
```

Claude가 대화형으로 Skill의 목적, 트리거 조건, 동작 방식을 물어보고 자동으로 SKILL.md를 생성해줍니다.

---

### 실전 예시: 메모리 최적화 Skill 만들기

**배경:** Claude 공식 문서에 따르면 MEMORY.md는 200줄 이하, 각 항목은 150자 이하로 유지해야 합니다. 이 규칙을 자동으로 점검하는 Skill을 만들어봅니다.

**1단계 — skill-creator 호출:**
```
/skill-creator

나는 Claude Code 메모리 파일(MEMORY.md, .md 파일들)을 공식 원칙에 따라
진단하고 최적화해주는 Skill을 만들고 싶어.
공식 문서 기반으로 체크해야 할 항목들:
- MEMORY.md 200줄 이하
- 각 인덱스 항목 150자 이하
- 중복 메모리 탐지
- 코드 패턴/파일경로를 메모리에 저장한 경우 제거 권고
- 오래된(stale) 메모리 탐지
```

**2단계 — 생성된 Skill 확인 및 사용:**
```
/memory-optimize
```

Claude가 현재 메모리 디렉토리를 분석하고, 문제 항목을 리스트업한 뒤 최적화 제안을 줍니다.

---

### 팀 공통 Skill 예시 (UniDocu6용)

skill-creator로 만들어두면 유용한 Skill들:

**1. AbstractJAVAService 코드 리뷰 Skill:**
```
/skill-creator

AbstractJAVAService를 상속한 Java 서비스 파일을 리뷰하는 Skill.
체크 항목:
- NSParam으로 입력받는지
- NSReturn으로 결과 반환하는지
- RFC 호출이 JCOManagerWrapper를 통해서만 이루어지는지
- 서비스 ID가 클래스명에서 "Service" 제거한 형태인지
```

**2. $nst 호출 패턴 점검 Skill:**
```
/skill-creator

JS 파일에서 $nst 호출 패턴을 점검하는 Skill.
체크 항목:
- is_data_ot_data(서비스명, params, callback) 형식 준수
- 동기 호출(콜백 생략) 남용 여부
- 에러 핸들링 누락 여부
```

**3. 커밋 메시지 생성 Skill:**
```
/skill-creator

변경된 파일과 이슈 번호를 받아서 "refs #이슈번호 [설명]" 형식의
커밋 메시지를 자동 생성하는 Skill.
```

---

### 커스텀 Skill 직접 작성

skill-creator 없이 직접 작성할 때의 기본 구조:

```markdown
---
name: ns-review
description: AbstractJAVAService 기반 서비스 파일 리뷰. "서비스 리뷰", "NS 리뷰" 언급 시 활성화.
allowed-tools: Read Grep Glob
---

$ARGUMENTS 파일(또는 최근 수정된 서비스 파일)을 리뷰하세요.

1. NSParam/NSReturn 사용 여부 확인
2. JCOManagerWrapper를 통한 RFC 호출 여부 확인
3. 서비스 ID 명명 규칙 확인 (클래스명 - "Service")
4. 문제 발견 시 파일명:라인번호 포함하여 리포트
```

**호출:**
```
/ns-review src/main/java/com/unipost/unidocu/ns/efi/ExpenseService.java
```

## 8. Plugin 활용

### Plugin이란?

Plugin은 Skills + Agents + Hooks + MCP 서버를 하나의 단위로 묶어서 설치/공유할 수 있는 패키지입니다.

```
Plugin = Skills + Agents + Hooks + MCP 서버 → 한 번에 설치
```

### 현재 사용 중인 주요 Plugin

**superpowers (강력 추천)**

가장 중요한 Plugin입니다. 코딩 워크플로우 전반을 강화하는 Skills 묶음:

| Skill | 언제 쓰나 |
|-------|-----------|
| `superpowers:brainstorming` | 새 기능 설계 전 아이디어 구체화 |
| `superpowers:writing-plans` | 멀티파일 수정 전 구현 계획 수립 |
| `superpowers:systematic-debugging` | 버그 발생 시 체계적 원인 분석 |
| `superpowers:verification-before-completion` | 작업 완료 전 검증 체크리스트 |
| `superpowers:requesting-code-review` | PR 전 코드 리뷰 요청 |

사용 예시:
```
새로운 AI 추천 기능을 추가하려는데 어떻게 설계하면 좋을지 brainstorming 해줘.
```
→ Claude가 자동으로 `superpowers:brainstorming` skill을 호출합니다.

---

**skill-creator (Skill 제작)**

대화형으로 새 Skill을 만들어주는 Plugin. 7장에서 설명한 대로 `/skill-creator`로 호출합니다.

```
/skill-creator
새로운 SAP RFC 연동 코드를 생성해주는 Skill을 만들어줘.
```

---

**frontend-design (UI 개발)**

고품질 프론트엔드 UI를 설계/구현할 때 사용:

```
/frontend-design
UD_0201_100.js 화면의 그리드 레이아웃을 개선하고 싶어.
현재 계정과목 입력 영역이 좁아서 사용성이 떨어져.
```

---

### Plugin 설치

```bash
# 마켓플레이스 탐색
/plugin

# 특정 Plugin 설치 (마켓플레이스에서)

```

### 코드 인텔리전스 Plugin (추천)

Java, JavaScript 등 언어별 LSP(Language Server Protocol) Plugin으로 Claude에게 **정확한 심볼 탐색** 능력 부여:

```bash

/plugin

# 설치 후 효과
- AbstractJAVAService 구현체 정확한 탐색

- 편집 후 자동 오류 감지
```

| `Esc` | 작업 중단 (컨텍스트 유지) |
| `Esc+Esc` | Rewind 메뉴 열기 |
| `Ctrl+G` | Plan Mode에서 계획을 편집기로 열기 |
| `Ctrl+O` | 상세 모드 토글 (추론 과정 보기) |
| `Alt+T` | 확장 사고 켜기/끄기 |
| `↑` | 명령 히스토리 |
| `Tab` | 명령 자동완성 |
| `?` | 사용 가능한 단축키 목록 |

### 세션 관리

```bash
claude --continue        # 가장 최근 세션 이어서
claude --resume          # 세션 선택 UI
claude -n session-name   # 이름으로 세션 시작
/rename session-name     # 현재 세션 이름 지정
```
