## 1. Claude Code란?

Claude Code는 Anthropic이 만든 **에이전트 코딩 도구**입니다. 단순히 질문에 답하는 챗봇이 아니라, 실제로 파일을 읽고, 코드를 수정하고, 명령을 실행하며, 문제를 자율적으로 해결하는 AI 개발 파트너입니다.

```
Claude Code = AI 두뇌 + 파일 읽기/편집 + 명령 실행 + Git 연동 + 외부 도구 연결
```

**핵심 특징:**
- 코드베이스 전체를 이해하고 여러 파일에 걸친 작업 수행
- 파일 읽기/편집, 터미널 명령 실행, Git 작업 자동화
- 자율적으로 계획하고, 구현하고, 검증
- 여러분이 자리를 비운 동안에도 작업 수행 가능

### 사용 가능한 환경

| 환경 | 특징 |
|------|------|
| **터미널 CLI** | 가장 완전한 기능, 파이프/스크립트 연동 |
| **VS Code 확장** | 인라인 diff, @-멘션, Plan 검토 |
| **JetBrains 플러그인** | IntelliJ IDEA, WebStorm 등 지원 |
| **데스크톱 앱** | 시각적 diff, 병렬 세션, 예약 작업 |
| **웹 (claude.ai/code)** | 브라우저에서 로컬 설치 없이 사용 |

## 2. 설치 및 시작하기

### 설치

**macOS / Linux**
```bash
npm install -g @anthropic-ai/claude-code
```

**Windows (PowerShell)**
```powershell
irm https://claude.ai/install.ps1 | iex
```

> Windows는 [Git for Windows](https://git-scm.com/downloads/win) 사전 설치 필요

### VS Code 확장 설치 (권장)

1. VS Code 확장 탭 열기 (`Ctrl+Shift+X`)
2. `Claude Code` 검색 후 설치
3. 사이드바 또는 `Ctrl+Shift+P` → `Claude Code: Open` 으로 실행

### JetBrains 플러그인 설치

1. Settings → Plugins → Marketplace
2. `Claude Code` 검색 후 설치
3. 우측 사이드바 Claude 아이콘으로 실행

### 첫 실행

```bash
# 프로젝트 디렉토리로 이동
cd /path/to/your/project

# Claude Code 시작 (첫 실행 시 로그인 진행)
claude
```

## 3. 빠른 시작 체크리스트

```
□ 1. Claude Code 설치 및 로그인
□ 2. 프로젝트 디렉토리에서 `claude` 실행
□ 3. "이 프로젝트의 구조를 설명해줘" 로 구조 파악
□ 4. /init 으로 CLAUDE.md 초안 생성 및 팀 규칙 추가
□ 5. .claude/settings.json 생성 (빌드 명령 허용, 운영 설정 파일 차단)
□ 6. IDE 연동 (VS Code/JetBrains Extension 설치)
□ 7. 일상 개발 작업에 Claude Code 적용 시작
```

> 팀 전체 도입 시: CLAUDE.md를 git에 커밋하여 팀원 전체가 동일한 컨텍스트를 가지도록 합니다.

## 4. 필수 명령어 & 단축키

### CLI 명령어

| 명령어 | 설명 |
|--------|------|
| `claude` | 대화형 세션 시작 |
| `claude "fix the login bug"` | 일회성 작업 실행 |
| `claude -p "explain this"` | 응답 후 종료 |
| `claude -c` | 이전 세션 이어서 |
| `claude -r` | 세션 선택하여 재개 |

### 슬래시 명령어

| 명령어 | 설명 |
|--------|------|
| `/help` | 도움말 |
| `/clear` | 대화 컨텍스트 초기화 |
| `/compact` | 대화 압축 (중요한 부분 유지) |
| `/init` | CLAUDE.md 자동 생성 |
| `/memory` | 메모리 파일 목록 확인 |
| `/model` | 모델 전환 |
| `/effort` | 추론 노력 수준 조정 |
| `/status` | 현재 상태 (모델, 계정 등) |
| `/btw` | 사이드 질문 (히스토리에 포함 안 됨) |
| `/rewind` | 이전 상태로 되돌리기 |

### 키보드 단축키

| 단축키 | 동작 |
|--------|------|
| `Shift+Tab` | Normal → Auto → Plan Mode 전환 |
| `Ctrl+C` | 현재 작업 중단 |
| `Esc` | 작업 중단 (컨텍스트 유지) |
| `Esc+Esc` | Rewind 메뉴 열기 |
| `Ctrl+G` | Plan Mode에서 계획을 편집기로 열기 |
| `↑` | 명령 히스토리 |
| `Tab` | 명령 자동완성 |

## 5. 핵심 워크플로우

### 컨텍스트 참조 문법

Claude에게 파일이나 조건을 직접 전달하는 두 가지 핵심 문법입니다:

```
@파일경로   → 해당 파일을 컨텍스트에 포함
# 조건/제약 → 인라인 주석으로 제약 조건 명시
```

**예시:**
```
@src/main/java/com/example/service/OrderService.java 이 서비스를 리팩토링해줘.
# 단, 기존 인터페이스와 반환 타입은 바꾸지 말 것
```

---

### 코드베이스 탐색

처음에 전체 맥락을 잡는 것이 중요합니다:

```
이 프로젝트가 어떤 구조로 되어 있는지 설명해줘.
```
```
주요 데이터 흐름을 프론트엔드부터 백엔드까지 설명해줘.
```
```
@src/main/java/com/example/service/OrderService.java 이 서비스의 역할을 설명해줘.
```

---

### 버그 수정

```
로그인 처리 중 세션이 유지되지 않는 버그가 있어.
에러 메시지: [에러 내용 붙여넣기]
@src/main/java/com/example/controller/AuthController.java 와
@src/main/webapp/js/login.js 를 확인하고 원인을 찾아줘.
```

**증상이 아닌 근본 원인을 설명하세요:**

| 모호한 요청 | 구체적인 요청 |
|------------|-------------|
| "그리드 버그 수정해줘" | "order.js에서 수량 수정 시 합계가 갱신되지 않음. onCellEdit 핸들러 확인 후 수정해줘." |

**에러 로그를 직접 파이프할 수도 있습니다:**
```bash
cat error.log | claude -p "이 에러의 근본 원인을 분석해줘."
```

---

### 새 기능 개발

**소규모 수정 (바로 구현):**
```
@src/main/webapp/js/order.js
저장 버튼 클릭 시 확인 다이얼로그를 추가해줘.
```

**중규모 기능 (Plan Mode 활용 권장):**
```
[Plan Mode로 전환 후]
주문 목록에 엑셀 다운로드 기능을 추가하고 싶어.
현재 OrderController.java와 order.js를 분석하고 구현 계획을 세워줘.
```

**Plan Mode 전환:** `Shift+Tab` 반복 → `⏸ plan mode on` 표시될 때까지

## 6. CLAUDE.md 기본 설정

### CLAUDE.md란?

Claude Code가 **매 세션 시작 시 자동으로 읽는** 마크다운 파일입니다. 프로젝트 규칙, 코딩 표준, 빌드 명령 등을 작성해두면 Claude가 항상 그 맥락을 가지고 작업합니다.

```
CLAUDE.md = Claude에게 보내는 지속적인 지시서
```

### `/init` 으로 자동 생성

```bash
# 프로젝트 루트에서 실행하면 코드베이스를 분석해서 CLAUDE.md 초안 생성
/init
```

### 파일 배치 위치

| 범위 | 위치 | Git 공유 |
|------|------|----------|
| **전역 (모든 프로젝트)** | `~/.claude/CLAUDE.md` | ❌ 개인 유지 |
| **프로젝트 (팀 공유)** | `./CLAUDE.md` | ✅ 커밋 권장 |
| **프로젝트 (개인)** | `./CLAUDE.local.md` | ❌ gitignore 추가 |

### CLAUDE.md 작성 핵심 원칙

- **200줄 이하** 유지 (길수록 준수율 저하)
- **구체적으로** 작성 ("코드 정리" ❌ → "DTO/VO 클래스 사용 필수" ✅)
- **자주 검토**하여 오래된/충돌하는 지침 제거

### 예시

```markdown
## 빌드 및 실행
- 빌드: `./gradlew build`
- 서버 실행: `./gradlew bootRun`
- 테스트: `./gradlew test`

## 코딩 규칙
- 외부 API 호출 결과는 반드시 DTO로 받아 변환 후 사용
- DB 트랜잭션: 서비스 레이어에서만 @Transactional 선언
- 커밋 메시지 형식: `refs #이슈번호 [모듈명] - [변경 내용]`

## 주의사항
- .env.production 파일은 절대 수정하지 말 것
- 외부 API 응답 파싱 시 null 체크 필수
```

## 7. 핵심 모범 사례

### 검증 방법을 함께 제공하라

Claude가 스스로 검증할 수 있을 때 결과 품질이 향상됩니다:

```
getOrderList API를 호출해서 목록 반환하도록 수정해줘.
빌드 후 서버 기동해서 실제로 확인해줘.
```

### 탐색 → 계획 → 구현 순서를 지켜라

```
잘못된 방식: "기능 추가해줘" → 즉시 코드 작성
올바른 방식: 탐색(Plan Mode) → 계획 검토 → 구현
```

### 컨텍스트를 초기화하며 작업하라

관련 없는 작업 사이에는 반드시 컨텍스트를 정리합니다:

```bash
/clear          # 컨텍스트 완전 초기화
/compact        # 중요한 내용은 유지하며 압축
/compact "API 연동 변경사항만 유지"
```

### 작업 규모에 맞게 접근하라

| 규모 | 접근 방식 |
|------|---------|
| 오타 수정, 로그 추가 | Plan Mode 없이 바로 진행 |
| 단일 기능 수정 | Plan Mode로 계획 후 구현 |
| 신규 기능, 리팩토링 | 탐색 → 계획 검토 → 단계적 구현 |

## 8. 트러블슈팅

### Claude가 같은 실수를 반복할 때

```
원인: 잘못된 시도들이 컨텍스트를 오염시킴
해결: 2번 연속 실패 시 → /clear 후 더 구체적인 초기 프롬프트로 재시작
```

### 응답 품질이 점점 저하될 때

```
원인: 컨텍스트 윈도우가 가득 참
해결: /compact 로 압축하거나, /clear 로 초기화 후 핵심 맥락만 다시 제공
```

### 원치 않는 변경이 일어났을 때

```bash
Esc+Esc     # Rewind 메뉴 → 이전 상태로 되돌리기
/rewind     # 동일한 효과
```

> Rewind는 Claude의 작업을 되돌리지만, 이미 실행된 shell 명령은 되돌릴 수 없습니다.

### Claude가 무한히 파일을 탐색할 때

```
원인: 범위가 너무 넓은 요청
해결: @파일명 으로 탐색 범위를 명시적으로 제한
      또는 "OrderService.java와 OrderController.java만 확인해줘" 처럼 범위를 지정
```

### 빌드/테스트 실패 후 Claude가 멈출 때

```
해결: 에러 메시지를 직접 복사해서 "이 에러로 실패함: [에러]. 수정하고 빌드 성공 확인해줘." 로 재지시
```

### 권한 요청이 너무 자주 뜰 때

자주 사용하는 명령어를 `.claude/settings.json`에 미리 허용합니다:

```json
{
  "permissions": {
    "allow": [
      "Bash(git *)",
      "Bash(./gradlew *)",
      "Bash(npm *)"
    ]
  }
}
```

> 더 자세한 설정 방법은 [심화 가이드](./클로드코드_사용가이드_심화.md)를 참고하세요.
